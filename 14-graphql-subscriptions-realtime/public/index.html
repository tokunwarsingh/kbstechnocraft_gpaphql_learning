GraphQL Mutations & Subscriptions

This page demonstrates different types of GraphQL mutations and real-time Subscriptions.

1. Create Mutation
Type: Create Operation

Creates a new country in the data.

mutation {
  createCountry(input: {
    code: "XX"
    name: "Test Country"
    capital: "Test City"
    currency: "TST"
    phone: "999"
  }) {
    code
    name
    capital
  }
}
2. Update Mutation
Type: Update Operation

Updates an existing country's information.

mutation {
  updateCountry(code: "XX", input: {
    name: "Updated Test Country"
    capital: "Updated City"
  }) {
    code
    name
    capital
  }
}
3. Delete Mutation
Type: Delete Operation

Deletes a country from the data.

mutation {
  deleteCountry(code: "XX")
}
4. Create Continent
Type: Create Operation

Creates a new continent.

mutation {
  createContinent(input: {
    code: "YY"
    name: "Test Continent"
  }) {
    code
    name
  }
}
5. Create State
Type: Create Operation

Creates a new state for a country.

mutation {
  createState(input: {
    code: "TS"
    name: "Test State"
    countryCode: "US"
  }) {
    code
    name
  }
}
6. Update State
Type: Update Operation

Updates a state's name.

mutation {
  updateState(countryCode: "US", code: "TS", name: "Updated Test State") {
    code
    name
  }
}
7. Delete State
Type: Delete Operation

Deletes a state from a country.

mutation {
  deleteState(countryCode: "US", code: "TS")
}
8. Multiple Mutations
Type: Batch Operations

Performs multiple mutations in one request.

mutation {
  createCountry: createCountry(input: {
    code: "ZZ"
    name: "Another Test Country"
    capital: "Another City"
    currency: "ZZZ"
    phone: "000"
  }) {
    code
    name
  }
  createContinent: createContinent(input: {
    code: "ZZC"
    name: "Another Test Continent"
  }) {
    code
    name
  }
}

Note: Open GraphiQL at http://localhost:4000/graphql to test these mutations.
