Understanding GraphQL Resolvers

This page demonstrates different types of GraphQL resolvers using the JSON data.

1. Root Query Resolver
Type: Query Resolver

Resolves top-level queries like continents, countries, etc.

query {
  continents {
    name
  }
}
2. Query Resolver with Arguments
Type: Query Resolver

Resolves queries that take arguments like continent(code: "AF").

query {
  continent(code: "AF") {
    name
  }
}
3. Field Resolver
Type: Field Resolver

Resolves fields on types, like countries field on Continent type.

query {
  continents {
    name
    countries {
      name
    }
  }
}
4. Computed Field Resolver
Type: Field Resolver

Resolves computed fields like countryCount that don't exist in the data.

query {
  continents {
    name
    countryCount
  }
}
5. Reverse Relationship Resolver
Type: Field Resolver

Resolves relationships in reverse, like finding continent from country.

query {
  country(code: "US") {
    name
    continent {
      name
    }
  }
}
6. Search Resolver
Type: Query Resolver

Resolves complex queries like searching/filtering data.

query {
  searchCountries(search: "united") {
    name
    capital
  }
}
7. Nested Field Resolver
Type: Field Resolver

Resolves deeply nested fields with their own resolvers.

query {
  continents {
    name
    countries {
      name
      stateCount
      states {
        name
      }
    }
  }
}

Note: Open GraphiQL at http://localhost:4000/graphql to test these resolvers. Check the server console for resolver execution logs.
